STEP 6: Correct Database Structure

This is VERY IMPORTANT.

Your website expects these nodes:

{
  "energy_monitor": {
    "voltage": 230,
    "current": 2.15,
    "power": 495,
    "energy_kWh": 6.82,
    "bill_INR": 54.56,
    "relay": 1
  },

  "energy_logs": {
    "log1": {
      "timestamp": 1776831000000,
      "voltage": 230,
      "current": 2.15,
      "power": 495,
      "energy_kWh": 0.32,
      "bill_INR": 2.56
    },

    "log2": {
      "timestamp": 1776834600000,
      "voltage": 228,
      "current": 1.98,
      "power": 451,
      "energy_kWh": 0.28,
      "bill_INR": 2.24
    }
  }
}
Meaning of Each Field
energy_monitor = Live Current Reading

Updated every few seconds.

energy_monitor

Used in:

Dashboard live cards
Live meter page
Bill page
energy_logs = History Records

Every 5 mins / 15 mins / 1 hour store logs.

Used in:

Daily bars
Weekly bars
Utility analytics
